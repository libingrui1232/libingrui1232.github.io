使用bind在linux上实现DNS正反向代理

系统运维
发布日期:   2021-07-19

DNS概述

一个地址 www.baidu.com. www是主机名 baidu.com 是域名 .是根域访问时省略

域与DNS的概念

​ 根域：位于树状结构最顶层，用”.”表示顶级域：一般代表一种类型的组织机构或国家地区；如.net(网络供应商)、.com(工商企业)、.org（团体组织）、.edu（教育结构）、.gov(政府部门)、.cn（中国国家域名）二级域：用来标明顶级域内的一个特定的组织，国家顶级域下面的二级域名由国家部门统一管理子域：二级域下所创建的各级域统称为子域，各个组织或用户可以自由申请注册自己的域名主机：主机位于域名空间最下层，就是一台具体的计算机域名与IP地址之间是多对一的关系，一个IP地址不一定只对应一个域名，且一个域名只可以对应一个IP地址

DNS服务器类型

(1)递归查询

​ 递归查询是一种DNS 服务器的查询模式，在该模式下DNS服务器接收到客户机请求，必须使用-一个准确的查询结果回复客户机。如果DNS服务器本地没有存储查询DNS信息,那么该服务器会询问其他服务器，并将返回的查询结果提交给客户机。(问你 你没有你帮我去办，在给我)

(2) 迭代查询

​ DNS服务器另外-种查询方式为迭代查询,当客户机发送查询请求时，DNS服务器并不直接回复查询结果,而是告诉客户机另- -台DNS服务器地址,客户机再向这台DNS服务器提交请求，依次循环直到返回查询的结果为止。(问你 你没有你让我找别人办)

利用bind配置DNS的正反向解析

正向解析 域名>>>IP 反向解析 IP>>>域名安装bind

安装 bind bind-chroot bind-utils

yum -y install bind bind-chroot bind-utils 
#centos的本地yum源没有bind-utils包所以我们要利用网络yum源，当然你不能利用centos6的yum源，因为centos6的yum源官方不支持服务了，当然不用bind-utils也行，他提供nslookup的命令
bind的配置目录 /etc/named.conf  /etc/named.rfc1912.zones  /var/named
vim /etc/named.conf #配置文件

vim /etc/named.rfc1912.zones # 配置正反解析文件
zone "text.com" IN {
        type master; #master是主DNS服务器的意思
        file "text.com.zone"; #ip与域名的对应关系的储存文件
        allow-update { none; }; #none值所有的ip都可以访问
}; #记得要加;好不然报错
# 正向解析文件 text.com是你自定的域名，当然什么都可以
# 反向解析文件
zone "109.168.192.in-addr.arpa" IN {   #你的ip要倒着写
        type master;
        file "192.168.109.zone"; #定义一个数据库文件
        allow-update { none; };
};
#这个配置文件有模板的

配置单独的数据库文件

正向解析文件

配置件文件介绍：

​ 1) SOA资源记录每个区在区的开始处都包含了一-个起始授权记录(Start of Authority Record) ,简称SOA记录。SOA定义了域的全局参数，进行整个域的管理设置。-一个区域文件只允许存在唯一-的SOA 记录。

​ 2) NS资源记录NS (Name Server)记录是域名服务器记录,用来指定该域名由哪个DNS服务器来进行解析。每个区在区根处至少包含- -个NS 记录。

​ 3) A资源记录地址(A) 资源记录把FQDN映射到IP地址。因为有此记录, 所以DNS服务器能解析FQDN域名对应的IP地址。

​ 4) PTR资源记录相对于A资源记录，指针(PTR) 记录把IP地址映射到FQDN。用于反向查询, 通过IP地址,找到域名。

​ 5) CNAME资源记录别名记录(CNAME)资源记录创建特定FQDN的别名。用户可以使用CNAME记录来隐藏用户网络的实现细节,使连接的客户机无法知道真正的域名。反向解析文件

反向解析文件

验证：将windows的DNS地址改为你的DNS服务器地址 输入nslookup

配置主从同步与区域访问控制

主从同步就是DNS服务器的冗余，一个是主DNS服务器，另一个是辅助DNS，在bind开数据同步，即可给辅助DNS服务器，同步数据。

配辅助DNS服务器
vim /etc/named.conf #改成any
vim /etc/named.rfc1912.zones
# 整向同步
zone "baidu.com" IN {
  type slave;                   #slave是辅助DNS服务器的意思
  file "slave/baidu.com.zone";  #文件设置同步
  masters { 192.168.109.138; }; #设置同步那个DNS服务器
};
# 反向同步
zone "109.168.192.in-addr.arpa" IN {
  type slave;  
  file "slave/192.168.109.zone";
  masters { 192.168.109.138; };
};
开启named服务测试成功

区域访问控制

​ 诺不配置named.rfc1912.zones文件，是个DNS服务器就可以同步资源，容易遭到暴力破解攻击，消耗服务器资源，所有我们可以实现定制DNS同步资源。

vim /etc/named.rfc1912.zones #配置文件 
将allow-update { none }; 改为 allow-transfer { 192.168.109.131; }; #这样只允许IP为192.168.109.131的dns服务器同步数据了

限制查询者

​ 限制某些IP询DNS服务器，好让黑客难以猜到DNS服务器地址，就是配置指定网段访问DNS服务器 

vim /etc/named.conf #设置网段

分离DNS

​ DNS的意思是一个域名根据网络可解析成不同IP、因为我们没有外网的环境，所有我们用一个局域网的不同IP代替

配置 /etc/named.conf 将监听端口改为any
将 named.conf里面的根域配置文件删除或注掉
#zone "." IN {
#  type hint;
#  file "named.ca";
#};
配置 vim/etc/named.rfc1912.zones
删除已有的配置，添加以下配置：
view "lan" {
        match-clients { 192.168.109.145; }; #匹配这个IP当然也可以匹配网段
        zone "zip.com" IN { #匹配这个IP，解析成下面的配置文件
          type master;
          file "zip.com.lan"; #定义的数据库文件
        };
        #根域文件
        zone "." IN {
          type hint;
          file "named.ca";
        };
};
view "wan" {
        match-clients { 192.168.109.132; };
        zone "zip.com" IN {
          type master;
          file "zip.com.wan";
        };
};

DNS域名转发

​ 假如一个域名是 baidu.com 它存在192.168.109.100的DNS服务器内，我们访问它会跳转到另一个DNS服务器的php.com的域名上去，而显示的域名还显示baidu.com

预备知识：递归查询与迭代查询

​ 想解析一个IP或者域名，我问你，你没有，你帮我问，然后你在给我答案，我问你，你没有，你给我一个地址，你让我去问他。前者好比递归查询，后者好比迭代查询。

实验环境：我们有以下的DNS地址：

Centos7 192.168.109.100 text.com （主）

Centos6 bak 192.168.109.149 baidu.com

Centos6 192.168.109.100 text.com

Centos7 192.168.109.147 zip.com

​ 我们是递归查询：目标我的机子问 100DNS 服务器他可以帮我转发请求，然后让我得到全部地址。

配置 /etc/named.conf
options {
        listen-on port 53 { any; };
        listen-on-v6 port 53 { ::1; };
        directory       "/var/named";
        dump-file       "/var/named/data/cache_dump.db";
        statistics-file "/var/named/data/named_stats.txt";
        memstatistics-file "/var/named/data/named_mem_stats.txt";
        recursing-file  "/var/named/data/named.recursing";
        secroots-file   "/var/named/data/named.secroots";
        allow-query     { any; };
        forwarders { 192.168.109.131;   #我没有的地址我会将解析发到以下地址
                     192.168.109.143; }; ##注意这里转发的是DNS地址，没有指定DNS转发域名。
  recursion yes; #是否使用递归式 DNS 服务器，默认为 yes

        dnssec-enable yes;
        dnssec-validation no;

        /* Path to ISC DLV key */
        bindkeys-file "/etc/named.root.key";

        managed-keys-directory "/var/named/dynamic";

        pid-file "/run/named/named.pid";
        session-keyfile "/run/named/session.key";
};
logging {
        channel default_debug {
                file "data/named.run";
                severity dynamic;
        };
};

zone "." IN {
        type hint;
        file "named.ca";
};

zone "baidu.com" IN {   #在访问baidu.con域名时自动跳转到forwarders
        type forward;   #定义的DNS服务器上去。
        forwarders { 192.168.109.149; };
};
include "/etc/named.rfc1912.zones";
include "/etc/named.root.key";  
#访问baidu.com域名时,将解析请求转到192.168.51.149(即grace.cn域名的DNS地址)，注意这个不能写在上面的forwarder处，否则转发无效！
#注意这里转发配置中制定了转发的域名，即forwad转发区(forward zone)设置，这是bind9之后的新特性
