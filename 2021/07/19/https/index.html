配置apache-https认证

系统运维
发布日期:   2021-07-19

使用openssl授权CA证书

​ 当然我们授权和申请的是私有的CA证书，CA对我们来说是https，https是密文访问网页协议

yum安装openssl

# 安装openssl软件 （环境Centos7） 我们使用网络yum源
yum -y install openssl openssl-devel
openssl的配置文件：
/etc/pki/
/etc/pki/tls
/etc/pki/tls/openssl.cnf

配置openssl.cnf文件

vim /etc/pki/tls/openss.cnf
#  配置ca的私钥和证书文件
# 更改certificate和private_key
[ CA_default ] #找到它，配置openssl所有的文件的对应关系
#我们要把文件名改的好记些（当然不改也行）改ca的私钥和证书文件
dir             = /etc/pki/CA           # 这是主配置文件目录，下面都是引用他
certs           = $dir/certs            # Where the issued certs are kept
crl_dir         = $dir/crl              # Where the issued crl are kept
database        = $dir/index.txt        # 数据的索引文件，增加签署文件的信息
#unique_subject = no                    # Set to 'no' to allow creation of
                                        # several ctificates with same subject.
new_certs_dir   = $dir/newcerts         # default place for new certs.

certificate     = /etc/pki/CA/private/ca.crt   # ca的证书
serial          = $dir/serial           # The current serial number
crlnumber       = $dir/crlnumber        # the current crl number
                                        # must be commented out to leave a V1 CRL
crl             = $dir/crl.pem          # The current CRL
private_key     = $dir/private/ca.key   # 私钥的 key
RANDFILE        = $dir/private/.rand    # private random number file

x509_extensions = usr_cert              # The extentions to add to the cert

# Comment out the following two lines for the "traditional"
# (and highly broken) format.
name_opt        = ca_default            # Subject Name options
cert_opt        = ca_default            # Certificate field options

更改签发地区

vim /etc/pki/tls/openssl.cnf
[ req_distinguished_name ] #找到它
countryName                     = Country Name (2 letter code)
countryName_default             = CN #中国
countryName_min                 = 2
countryName_max                 = 2

stateOrProvinceName             = State or Province Name (full name)
stateOrProvinceName_default     = BJ #北京

localityName                    = Locality Name (eg, city)
localityName_default            = BJ #北京

0.organizationName              = Organization Name (eg, company)
0.organizationName_default      = xiaoming #你的机构
# 加了 # 号的全添加
# 生成的文件的路径一定要按照配置的来，不然报错！如过你不想使用这个目录你当然可以自定义

配置签发文件

[root@localhost pki]# cd /etc/pki/CA/
[root@localhost CA]# ls
certs  crl  newcerts  private
[root@localhost CA]# touch index.txt #增加签署文件的信息
[root@localhost CA]# echo 00 > serial #自动增量

配置私钥与证书文件

cd /etc/pki/CA/private
### 生产私钥文件：
# 这里的文件名都是你在 openssl.cnf 定义的
openssl genrsa -out /etc/pki/CA/private/ca.key -des3 2048
让你定义私钥密码：rethat
### CA自签名生成CA证书（我自己给自己下发....）
openssl req -new -x509 -days 7300 -key /etc/pki/CA/private/ca.key > ca.crt # 7300 是天的意思，有效期为20年（也是最长期限）
属入密码：redhat就配置好了

httpd服务生成私钥及证书签名

安装 mod_ssl 包

yum -y install mod_ssl
openssl genrsa -out /etc/httpd/httpd.key #生产私钥
openssl req -new -key /etc/httpd/httpd.key -out /home/httpd.csr #生成证书签名请求
openssl ca -in /home/httpd.csr -out /home/httpd.crt #ca服务器签名
将 ca.crt文件复制到 /etc/httpd/ 目录下
httpd.csr是申请证书的请求 httpd.crt是ca签名的证书 httpd.key是ca验证的私钥
建立一个ssl目录在 /etc/httpd/httpd.d下
我们建立一个www.conf文件当作ssl的配置文件
cd /etc/httpd/conf.d
添加文件：
<VirtualHost 192.168.109.158:443>
	ServerName www.libingrui.com #这是拥有ca认证的网站
	DocumentRoot /var/www/html #根目录
	
	SSLEngine on #开启ssl加密
	SSLProtocol all -SSLv2 -SSLv3 #下面是加密算法
	SSLCipherSuite ALL:!ADH:!EXPORT:!SSLv2:RC4+RSA:+HIGH:+MEDIUM:+LOW
	
	SSLCertificateFile /etc/httpd/httpd.crt #链接到ca签名证书
	SSLCertificateKeyFile /etc/httpd/httpd.key #链接到ca的私钥
</VirtualHost>

强制使用https

​ 一个用户使用 www.baidu.com 访问我搭建的网站，默认还是http：80，要实现用户无论访问baidu.com还是 www.baidu.com 都自动跳转 https://baidu.com:443

vim /etc/httpd/conf.d/www.conf 添加：
<VirtualHost 192.168.109.131:80>
	DocumentRoot /var/www/html
	ServerName www.baidu.com
	ServerAlias baidu.com

	RewriteEngine on
	RewriteRule ^(.*)$ https://www.baidu.com$1 [R=301,L]
</VirtualHost>
ServerName www.baidu.com ServerAlias baidu.com 用户在访问 www.baidu.com和baidu.com时自动跳转到 https://www.baidu.com 上去
验证实验：

​

​ 思路先使用openssl申请ca的私钥和ca证书，在使用mod_ssl认证httpd的私钥，在生成提交证书签名请求，使用ca签署签名请求

​ 注：在ca生成证书和生成提交证书签名请求的Common Name字段都要填上网址，前者是ca的签署网址如ca.xiaomi.com，后者是可以使用ca签名的网址要你自定义如www.libingrui.com等
